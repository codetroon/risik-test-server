## Project overview

Backend API for the Political Analysis CRM ([docs/PRD.md](docs/PRD.md)) — a constituency-management and political-intelligence platform with four roles (`super_admin`, `admin`, `researcher`, `officer`) and strict security requirements (no third-party AI/cloud services, audit logging, rate-limited logins).

Stack: Node 22+, Express 5, TypeScript (strict, ESM), PostgreSQL via Prisma 7, Redis (rate limiting + password-reset tokens), Zod v4, pino, JWT + argon2 auth, Swagger UI via zod-openapi.

## Setup & commands

```bash
cp .env.example .env     # then fill values (JWT_SECRET must be ≥ 32 chars)
docker compose up -d     # Postgres :5432 + pgAdmin :5500 + Redis :6379 + RedisInsight :5540 (Redis + Insight loopback-only)
npm install
npm run db:generate      # Prisma Client → src/generated/prisma (gitignored)
npm run db:migrate       # create + apply dev migration (prompts for a name)
npm run db:seed          # seed the admin user

npm run dev              # dev server with hot reload (tsx watch src/server.ts)
npm run build            # tsc → dist/
npm start                # node dist/server.js
npm run db:studio        # Prisma Studio DB browser
npx tsc --noEmit         # type-check only
```

Full setup guide: [docs/SETUP.md](docs/SETUP.md).

## Testing

There is no test runner or linter configured yet. `npx tsc --noEmit` is the correctness check — run it before considering any change done.

## Code style & conventions

- **ESM + NodeNext**: every relative import uses a `.js` extension, even for `.ts` files (`import { env } from "./config/env.js"`).
- **Env access**: never read `process.env` directly — import `env` from `src/config/env.ts` (Zod-validated at startup). New env vars go in that schema **and** `.env.example`.
- **Response contract**: controllers never call `res.json`. Success goes through `responseHandler.ok/created/noContent/paginated` (`src/shared/lib/responseHandler.ts`); failures are thrown as `new ApiError.NotFound(...)`, `ApiError.Conflict(...)`, etc. (`src/shared/errors/AppError.ts`) and converted by the global error handler.
- **Validation**: Zod schemas in `<feature>.schemas.ts`, applied via the `validate({ body/query/params })` middleware. Derive TS types with `z.infer` — no hand-written interfaces that can drift.
- **Auth**: protect routes with `requireAuth` (`src/shared/middleware/requireAuth.ts`); it attaches `{ id, email, role }` to `req.auth`.

## Project structure

- `src/server.ts` → `src/app.ts` — entry point wrapping the middleware stack (helmet → cors → json → requestId → pino-http → routes → swagger → unknown-route → error handler).
- `src/modules/<feature>/` — feature modules, each with a fixed file set: `*.schemas.ts`, `*.routes.ts`, `*.controller.ts`, `*.service.ts`, `*.docs.ts`. Wiring a new module = add its router to `src/modules/index.ts` (mounted under `/api/v1`) + its docs to `src/shared/docs/openapi.ts`.
- `src/shared/` — middleware, errors, response/logger libs, OpenAPI builder.
- `prisma/schema.prisma` — generator/datasource only; models live in `prisma/model/*.prisma` (Prisma 7 multi-file schema, CLI pointed at `prisma/` by `prisma.config.ts`).
- `src/db/prisma.ts` — PrismaClient singleton (pg driver adapter). Always import this, never instantiate a new client.

API docs: Swagger UI at `/api/v1/docs` (non-production only). When documenting a module, follow [docs/Swagger-guide.md](docs/Swagger-guide.md).

## Boundaries

- Never commit `.env` or secrets.
- Never edit `src/generated/` — Prisma output, regenerated by `db:generate`.
- Never call third-party AI or cloud services from application code — the PRD forbids off-premise data transmission without client permission.
- Schema changes go through `npm run db:migrate` — never edit applied migration files in `prisma/migrations/`.
- Redis (`src/config/redis.ts` singleton) backs rate limiting and password-reset tokens; the server fails fast at startup if Redis is unreachable, but the rate limiter fails open on runtime Redis errors (by design — an outage must not block logins).
- Deployments behind a reverse proxy **must** set `TRUST_PROXY` (hop count, e.g. `1` for a single nginx) or every client shares the proxy's IP for rate limiting.

## Git

- Conventional commits, matching existing history: `feat(scope):`, `fix:`, `refactor:`, `docs:`, `chore:`.
- Run `npx tsc --noEmit` before committing.
